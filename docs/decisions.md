# 설계 의사결정 기록 (ADR)

## 1. 무엇을 만드는가
세무대리인이 매월(또는 반기) 인건비 신고 3종 —
**① 홈택스 원천세 파일변환신고 ② 위택스 지방소득세 특별징수 신고 ③ 홈택스 간이지급명세서 제출** —
과 그 뒤 서류처리(접수증·신고서·납부서 PDF/출력)를 자동화하는 Windows 데스크톱 프로그램.
변환파일은 사용자가 **세무사랑**으로 미리 생성.

## 2. 스택·엔진 — yangdo_auto 그대로 재사용
같은 사무소의 성공·배포된 `yangdo_auto`(양도세: 홈택스 파일변환신고 + 위택스 회계파일신고)와
**동일 스택(Python+Playwright+Tkinter), 동일 phase 구조, 동일 기계장치**를 쓴다.
- 그대로 복사(검증된 코드): `browser.py`(영구프로필+kiosk+PDF sticky), `util.py`,
  `pdf_save.py`(pywinauto OS다이얼로그+mtime 검증), 파일변환신고 기계장치
  (파일주입/검증/모달/제출), clipreport(홈택스)·OZ뷰어(위택스) 출력, GUI 전체.
- 새로 확정할 것: 사이트 내 **내비게이션(메뉴 경로/URL)**과 화면별 차이 — `LIVE-TODO` 주석으로 표시.

근거: 홈택스 '전자파일변환' 화면은 세목 공통(버튼 텍스트 동일). 위택스 파일신고도
양도분과 같은 패턴(파일주입→다음→서식검증→제출)으로 추정. 다른 것은 진입 경로뿐.

## 3. Phase 구성 — 신고 3종 + 출력 4종
1. **위택스 특별징수 파일신고** — 가상계좌 지연 대비 먼저
2. **홈택스 원천세 파일변환신고**
3. **홈택스 간이지급명세서 제출** — 근로/사업 등 여러 파일이면 `;` 구분, 순차 제출
4. **홈택스 접수증·신고서 출력** / 5. **위택스 신고서 출력**
6. **홈택스 납부서 출력** / 7. **위택스 납부서 출력**(가상계좌 대기 후)

양도세와 달라진 점:
- 부속서류 제출 phase 없음(인건비 신고엔 불필요)
- 조회 키가 주민번호 → **사업자등록번호(10자리)** + 업체명
- 홈택스 납부서 파일명 세목: 양도소득세 → **원천세**
- `제출까지 자동` 토글을 GUI에 노출(초기 테스트 안전장치)

## 4. 미확정 사항 (라이브 확인으로 채울 것 — LIVE-TODO)
- 원천세 파일변환신고 메뉴 경로: `신고/납부 → 원천세 → 파일변환신고` 로 추정만 해둠
- 간이지급명세서 제출: 메뉴 경로·화면이 전자파일변환식인지 자체 제출화면인지 미확인
- 위택스 특별징수 파일신고/신고내역 URL: 비워둠(`WETAX_FILING_URL=""`) — 메뉴 텍스트 폴백으로 탐색
- **일괄 파일(여러 업체) 처리**: 세무사랑 변환파일에 여러 업체가 들어가면 신고 1회로 끝나지만,
  납부서·서류 출력은 업체별 반복이 필요 → 첫 라이브 테스트에서 파일/화면 구조 보고 결정.
  (현 GUI는 업체 1곳 기준 입력. 다건이면 업체 목록 CSV 입력 등으로 확장 예정.)
- 간이지급명세서 제출분(근로/사업/기타 인적용역)의 접수증 출력 흐름

## 5. 개발 방법 — yangdo_auto와 동일
phase 하나씩, 라이브 화면 보며 사용자가 단계별로 알려주는 대로 채운다.
순서 권장: ② 홈택스 원천세 파일변환신고(전자파일변환 화면이 검증된 기계장치와 같은지 확인)
→ ① 위택스 → ③ 간이지급명세서 → 출력 phase들.
첫 테스트는 반드시 `제출까지 자동` OFF로 검증 화면까지만.
